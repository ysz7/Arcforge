How updates work

Arcforge periodically checks a small JSON file hosted on GitHub Pages to see if a new version is available.

When the Arcforge desktop client starts, it reads its own version (for example 1.1.0) and performs a simple GET request to:

https://ysz7.github.io/Arcforge/updates.json

The file updates.json is a static manifest that describes the latest stable version, the minimum supported version, and one or more download links for Windows builds.

If a newer version is available, Arcforge shows a small prompt and lets you open the release or changelog in your browser. If you are already on the latest version, the client stays quiet.

No background services. The check is a simple HTTPS request to a static JSON file on GitHub Pages. There is no telemetry and no tracking – your code never leaves your machine.
// Pseudo-code for how Arcforge checks for updates
const currentVersion = "1.1.0";
const url = "https://ysz7.github.io/Arcforge/updates.json";

fetch(url)
  .then(res => res.json())
  .then(manifest => {
    // compare manifest.latestVersion with currentVersion
    // if newer: ask user and open changelogUrl or Releases
  });
